# AI 品牌命名工作流（细化版）

本文件是 `SKILL.md` 默认工作流的扩展，解决三个问题：
1. 每一步**具体怎么做**
2. 命名原型的细节和例子
3. 做 **naming agent / 起名网站 / prompt 设计** 时的骨架参考

主流程的 5 步定义在 SKILL.md。本文件补充每一步的实战细节。

---

## Step 1 · 定位 + 约束 的实战做法

当用户信息不足时，问题不要一口气抛 9 条，优先只问**最关键的 2-4 条**：

必问（几乎每次都要）：
- 这是什么产品 / 服务 / 品牌？卖什么？
- 目标用户是谁？
- 希望给人什么感觉？

看情况问：
- 偏中文 / 英文 / 国际化？
- 要不要考虑 `.com` / 商标？
- 想不想和某些品牌区别开（不想像谁）？
- 是否有偏好的风格（直接 vs 抽象、硬 vs 软、工具感 vs 生活感）？

**不要把 9 个问题整齐列一遍发给用户**——那是文档视角，不是对话视角。

---

## Step 2 · 语言空间发散的做法

### 关键词发散
围绕这些维度：
- 核心概念与功能
- 情感与体验
- 隐喻与象征
- 用户语言（用户怎么描述这个东西）
- 行动与过程

按需要整理成 20-80 个关键词（快速场景 20 个就够，完整方案再扩到 80），并按以下四类归组：
- 【核心名词】
- 【动态动词】
- 【形容词 / 感受】
- 【抽象概念 / 意象】

### 隐喻探索（先于生成候选）
见 `metaphor-mapping.md`。问一句：
**"这个品牌本质上像 ______"**
然后扩 5-10 个同域词，再在这些词里长名字。

## Step 3 · 命名原型的细节

### 描述型命名
直接讲品类 / 功能。传播成本低，独特性低。
- 参考：American Airlines、General Electric、每日优鲜

### 暗示 / 联想型命名
不直接描述，但暗示价值或场景。
- 参考：Airbnb（气床 + bnb）、滴滴（声音 + 动作感）、观夏

### 造词命名
创造新词，独特性极高，记忆成本略高。**避免落入 AI slop。**
- 参考：Kodak、Xerox、Figma、Notion、Stripe、Vercel

### 拼合命名（Portmanteau）
两个词合并。做得好很精彩，做得差就是 slop。
- 好：Pinterest（pin + interest）、Instagram（instant + telegram）
- 差：几乎所有以 `-ify / -ora / -exa` 结尾的硬拼

### 创始人命名
适合个人品牌、服务业、奢侈品、律所。
- 参考：Ford、Disney、老干妈、霸王

### 地名命名
依托地理文化资产。
- 参考：Amazon（河）、Patagonia（地区）、北冰洋、西湖龙井

### 首字母缩写（A.K.A. 初创期慎用）
通常是老牌大公司不得已的结果。新品牌起名**不推荐**一上来就用缩写。
- 参考：IBM、HSBC、BMW（都是后期简化）

### 组合命名法（中文强推荐）
这是提高可注册性和画面感的性价比最高的方法。

- **东西 + 动作**：飞猪、闲鱼
- **东西 + 颜色**：小红书、蓝瓶咖啡
- **东西 + 数字**：三只松鼠、六个核桃、三顿半
- **东西 + 东西**：农夫山泉、老乡鸡

## Step 4 · shortlist 评审的做法

### 用强制表格，不要自由散文
见 `assets/shortlist-template.md`。

### 五标准 + fatal flaw 双层打分
- 五标准 → 每项标 "高 / 中 / 低"，不要打 1-10 分（10 分制会让差异被平均掉）
- fatal flaw → 每个候选**必须**标一句"最可能死在哪"

### 否定指令（用户反馈不满意时）
收到"不满意"之后，优先补**否定约束**，不要盲目再生成：
- 避免宗教化词汇
- 避免廉价感词汇
- 避免互联网黑话
- 避免太像科技公司模板
- 避免英文造词味过重
- 避免与 XX / YY 品牌风格过近

## Step 5 · 收敛的做法

### 提问要"会改变结果"
不要问："你觉得哪个最好？" （这把判断甩回用户）
要问：
- "你更想要'清晰好传播'，还是'有品牌感但更抽象'？"
- "这个名字未来要走国际化吗？"
- "你希望名字是'品类入口'，还是'品牌资产'？"
- "预算允许买二手域名吗？"

### 融合方案
用户挑出 3-10 个喜欢的之后，基于共同方向做 3-5 个融合方案。
每个方案附：
- 创意来源 / 解读
- 核心优势
- 建议搭配的 slogan / tagline

## 做 Naming Agent / 起名网站时的骨架

如果用户不是来起一个名字，而是想做产品：

1. **不要做一键吐名字的产品**。那是 demo，不是工具。
2. 默认流程应该是 **定位问答 → 3 方向 → shortlist → 可调旋钮**
3. 名字页面里要有：理由、风险、可调性，而不是只有词面
4. 把本 SKILL.md 的 5 步流程抽成 agent 骨架，每步一个 state
5. "可调旋钮"和"否定指令"要作为独立 UI 控件暴露，不要藏在自由输入里

---

## 对 agent 的提醒

- 不要期望一步到位
- 善用"广度生成 + 深度判断"
- 第一轮先打开空间，第二轮再聚焦
- 真正有价值的不是一下中一个神名，而是**帮用户更快做出高质量决策**
