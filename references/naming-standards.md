# 品牌命名标准

## 一、什么是好名字

品牌名不是先有词，再找解释。
品牌名是定位战略之后，顺应得出的语言表达结果。

所以命名前先回答：
- 你到底卖什么
- 你要服务谁
- 你希望别人怎么记住你
- 你不希望自己像谁

如果这四件事都不清楚，后面很容易在一堆名字里迷路。

## 二、好名字的五个标准

### 1. 好记
- 容易发音
- 容易拼写
- 容易记住
- 避免生僻字、过长、拗口、难输入

参考：
- 苹果
- 小米

### 2. 相关
- 和品类、功能、场景、价值主张有关
- 不是完全无来由的“空中词”

参考：
- 饿了吗：和外卖、餐饮场景高度相关
- 三顿半：和三餐之外的咖啡联想相关

### 3. 有戏
- 能带出画面、故事、意境或正面情绪
- 能和品牌调性形成统一感

参考：
- 观夏：东方意境
- 元气森林：健康、活力、轻盈感

### 4. 能用
- 有注册、域名、账号可用潜力
- 这是非常现实的一条

注意：
未核实时只能写：
- [商标待核实]
- [域名待核实]
- [账号待核实]

### 5. 包容
- 不轻易把未来产品线锁死
- 能覆盖未来的扩展方向
- 和品类词组合时不违和

## 三、额外判断原则

### 1. 客观存在的东西更容易被记住
用“客观存在”的东西做品牌名，通常更容易降低记忆和传播成本。
因为人脑更容易检索熟悉事物。

参考：
- 小米
- 苹果

### 2. 越直接，不一定越普通
“直接”不等于“没品牌感”。
很多大众传播效率极高的名字，都来自简单、具体、可感知的对象。

### 3. 不要为了特别而特别
太怪、太绕、太像AI拼出来的名字，往往会提高沟通成本。

### 4. 域名不是技术问题，也是品牌信号问题
尤其对创业品牌 / SaaS / 英文产品名来说，主域名是否干净、是否容易混淆，会持续影响用户对品牌的第一印象。

现实判断：
- `.com` 拿不到，不一定马上判死刑
- 但如果 `.com` 被强势竞品占据，且你的名字本身也不够强，长期传播会很吃亏
- 不要默认“换个后缀就好了”，要判断这是合理妥协，还是应该认真考虑换名

### 5. 命名是独立技能，创始人容易对旧名字过度依恋
很多人会高估自己当前名字的质量，只因为它已经变成“我的项目”了。

遇到重命名时，要主动做这个思维实验：
- 如果这个名字不是你现在的名字，你还会这么坚持吗？
- 你坚持它，是因为它真的更好，还是因为你已经习惯了它？

## 四、评估表建议

评估候选名时，可用以下维度：

| 维度 | 说明 |
|---|---|
| 好记 | 是否顺口、易输、易写、易传播 |
| 相关 | 是否和品牌、品类、场景有关 |
| 有戏 | 是否有联想、画面、情绪价值 |
| 能用 | 是否有注册/域名/账号潜力（初步） |
| 包容 | 是否支持未来扩展 |
| 独特性 | 是否容易撞车或太像别人 |
| 风险 | 是否有歧义、负面联想、跨文化问题 |

## 五、常见误区

- 误区 1：以为越高级越好
- 误区 2：以为越抽象越有品牌感
- 误区 3：只看自己喜欢，不看传播成本
- 误区 4：只顾创意，不顾注册现实
- 误区 5：一上来就批量吐 100 个名字，没有判断标准

## 六、对 agent 的执行要求

当你评估品牌名时，不要只说“我觉得不错”。
要明确指出它在“五标准”上分别成立还是不成立。

如果用户陷入“都还行”的摇摆状态，就把讨论拉回标准，而不是继续无限加名字。
